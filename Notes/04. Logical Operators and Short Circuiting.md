# 📄 JavaScript Operators & Short-Circuiting — Frontend Interview Revision

These operators are extremely important in:

* React rendering
* State management
* Optional configs
* Default values
* API handling
* Conditional rendering

Interviewers LOVE edge cases here.

---

# 1️⃣ `||` (Logical OR)

Returns the **first truthy value**.

```js id="e11"
const value = a || b;
```

Meaning:

```text id="e12"
If a is truthy -> return a
Else -> return b
```

---

# ✅ Examples

```js id="e13"
0 || 10
// 10
```

```js id="e14"
"" || "hello"
// "hello"
```

```js id="e15"
false || "yes"
// "yes"
```

---

# 🚨 Important

`||` treats ALL falsy values as missing.

Falsy values:

```js id="e16"
false
0
""
null
undefined
NaN
```

---

# 🔥 Common Use

## Default values

```js id="e17"
const username = input || "Guest";
```

---

# 🚨 Problem with `||`

```js id="e18"
const count = 0 || 10;
```

Output:

```js id="e19"
10
```

But maybe `0` was valid.

---

# 2️⃣ `??` (Nullish Coalescing)

Returns right side ONLY if left side is:

* `null`
* `undefined`

---

# ✅ Example

```js id="e20"
0 ?? 10
// 0
```

```js id="e21"
"" ?? "hello"
// ""
```

```js id="e22"
null ?? "fallback"
// "fallback"
```

---

# 🔥 MOST IMPORTANT DIFFERENCE

| Operator | Treats as missing   |   |                  |
| -------- | ------------------- | - | ---------------- |
| `        |                     | ` | ALL falsy values |
| `??`     | only null/undefined |   |                  |

---

# 🚨 Frontend Rule

Use:

```js id="e23"
??
```

for:

* API defaults
* user input
* numeric values

Safer.

---

# 3️⃣ `&&` (Logical AND)

Returns first falsy value OR last truthy value.

---

# ✅ Examples

```js id="e24"
true && "hello"
// "hello"
```

```js id="e25"
0 && "hello"
// 0
```

```js id="e26"
"yes" && 123
// 123
```

---

# 🔥 React Conditional Rendering

Very common:

```jsx id="e27"
{isLoggedIn && <Dashboard />}
```

Meaning:

* render only if truthy

---

# 🚨 Dangerous Edge Case

```jsx id="e28"
{0 && <Component />}
```

Renders:

```text id="e29"
0
```

Because `0` is returned.

---

# ✅ Safer

```jsx id="e30"
{!!count && <Component />}
```

---

# 4️⃣ Optional Chaining `?.`

Prevents crashes on null/undefined access.

---

# ✅ Example

```js id="e31"
user?.profile?.name
```

Instead of:

```js id="e32"
user && user.profile && user.profile.name
```

---

# 🚨 Important

Only protects:

* null
* undefined

Not other falsy values.

---

# ✅ Optional Function Calls

```js id="e33"
callback?.()
```

---

# 5️⃣ Nullish Assignment `??=`

Assign only if null/undefined.

```js id="e34"
obj.value ??= 10;
```

---

# 6️⃣ OR Assignment `||=`

Assign if falsy.

```js id="e35"
obj.value ||= 10;
```

---

# 🚨 Difference

```js id="e36"
let x = 0;
x ||= 5;
// 5
```

```js id="e37"
let x = 0;
x ??= 5;
// 0
```

---

# 7️⃣ AND Assignment `&&=`

Assign only if current value truthy.

```js id="e38"
x &&= y;
```

---

# 8️⃣ Ternary Operator

```js id="e39"
condition ? a : b
```

---

# ✅ Example

```js id="e40"
const role = isAdmin ? "Admin" : "User";
```

---

# 🚨 Nested Ternaries

Avoid deep nesting.

Bad readability.

---

# 9️⃣ `==` vs `===`

---

# `===`

Strict equality.

NO coercion.

Always preferred.

---

# 🚨 `==` Weirdness

```js id="e41"
0 == false
// true
```

```js id="e42"
"" == false
// true
```

```js id="e43"
null == undefined
// true
```

---

# ✅ Interview Rule

Use:

```js id="e44"
===
```

almost always.

---

# 🔟 Truthy vs Falsy

---

# Falsy Values

ONLY these:

```js id="e45"
false
0
-0
0n
""
null
undefined
NaN
```

Everything else is truthy.

---

# 🚨 Weird One

```js id="e46"
[]
{}
```

are truthy.

---

# 1️⃣1️⃣ Short-Circuiting

---

# OR Short-Circuit

```js id="e47"
a || b
```

If `a` truthy:

* `b` never evaluated

---

# AND Short-Circuit

```js id="e48"
a && b
```

If `a` falsy:

* `b` never evaluated

---

# ✅ Useful Pattern

```js id="e49"
isReady && doSomething()
```

---

# 1️⃣2️⃣ Destructuring Defaults

```js id="e50"
const { name = "Guest" } = user;
```

Only applies if:

* undefined

NOT null.

---

# 1️⃣3️⃣ Default Parameters

```js id="e51"
function greet(name = "Guest") {}
```

Only for:

* undefined

---

# 🚨 Difference from `||`

```js id="e52"
greet("")
```

Keeps empty string.

---

# 🔥 Most Important Frontend Patterns

---

# Safe API Access

```js id="e53"
response?.data?.user
```

---

# Default State

```js id="e54"
const count = value ?? 0;
```

---

# Conditional Rendering



# Fallback UI

```js id="e57"
const title = apiTitle || "Untitled";
```

# ⚡ 2-Line Revision

> `||` returns the first truthy value while `??` only falls back for null/undefined, making it safer for frontend defaults.
> `&&` enables short-circuit conditional execution/rendering, heavily used in React and JS control flow.

* Truthy/falsy values
* Short-circuit evaluation

* Assignment operators (`||=`, `??=`)

# 🔥 Golden Memory Trick

```text id="e58"
||  -> first truthy
&&  -> first falsy
??  -> first defined
```

And:

```text id="e59"
Use ?? when 0/false/"" are valid values
```
