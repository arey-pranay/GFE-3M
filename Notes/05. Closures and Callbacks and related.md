# 1️⃣ Closures

## ✅ Definition

A closure is when a function remembers variables from its lexical scope even after the outer function has finished execution.

---

# ✅ Example

```js id="g11"
function outer() {
  let count = 0;

  return function inner() {
    count++;
    console.log(count);
  };
}

const increment = outer();

increment(); // 1
increment(); // 2
```

---

# 🧠 Why It Works

`inner()` still has access to:

```js id="g12"
count
```

because JS preserves the lexical environment.

---

# 🔥 Interview Definition

> Closures allow functions to retain access to variables from their outer scope even after the outer function has returned.

---

# 🚨 Most Important Uses

| Use Case       | Why                 |
| -------------- | ------------------- |
| Debouncing     | preserve timer      |
| Throttling     | preserve timestamps |
| React hooks    | state persistence   |
| Data privacy   | encapsulation       |
| Memoization    | cache retention     |
| Event handlers | remember context    |

---

# 🔥 Private Variables Pattern

```js id="g13"
function createCounter() {
  let value = 0;

  return {
    increment() {
      value++;
    },

    getValue() {
      return value;
    }
  };
}
```

`value` cannot be directly accessed outside.

---

# 🚨 Common Interview Trap

```js id="g14"
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i);
  }, 100);
}
```

Output:

```text id="g15"
3
3
3
```

Because:

* `var` is function scoped
* closure captures SAME variable

---

# ✅ Fix with `let`

```js id="g16"
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i);
  }, 100);
}
```

Output:

```text id="g17"
0
1
2
```

Because `let` creates new binding per iteration.

# 2️⃣ Callbacks

## ✅ Definition

A callback is a function passed into another function to be executed later.

---

# Example

```js id="g18"
function greet(name, callback) {
  console.log("Hello", name);

  callback();
}
```

---

# 🔥 Callback Hell

Nested async callbacks:

```js id="g19"
getUser(() => {
  getPosts(() => {
    getComments(() => {

    });
  });
});
```

Led to:

* Promises
* async/await

---

# 3️⃣ Higher Order Functions (HOF)

## Function that:

* takes function as argument
  OR
* returns function

---

# Examples

```js id="g20"
map
filter
reduce
debounce
throttle
```

---

# Example

```js id="g21"
function multiplyBy(x) {
  return function(y) {
    return x * y;
  };
}
```

---

# 4️⃣ Lexical Scope

Functions access variables based on where they are defined, NOT where called.

---

# Example

```js id="g22"
const name = "global";

function outer() {
  const name = "outer";

  function inner() {
    console.log(name);
  }

  inner();
}
```

Output:

```text id="g23"
outer
```

---

# 5️⃣ Execution Context

Whenever a function runs, JS creates:

* memory phase
* execution phase

Contains:

* variables
* scope chain
* `this`

---

# 6️⃣ Scope Chain

JS searches variables:

1. local scope
2. parent scope
3. global scope

# 7️⃣ Event Loop

One of MOST IMPORTANT frontend topics.

---

# JS is Single Threaded

Only one call stack.

Async tasks handled by:

* browser APIs
* queues
* event loop

---

# Simplified Flow

```text id="g24"
Call Stack
↓
Web APIs
↓
Callback Queue
↓
Event Loop
```

---

# Example

```js id="g25"
console.log(1);

setTimeout(() => {
  console.log(2);
}, 0);

console.log(3);
```

Output:

```text id="g26"
1
3
2
```

---

# 🚨 Why?

`setTimeout` goes to Web APIs first.

---

# 8️⃣ Microtasks vs Macrotasks

VERY IMPORTANT.

---

# Microtasks

Higher priority.

Examples:

* Promise.then
* queueMicrotask

---

# Macrotasks

Examples:

* setTimeout
* setInterval

---

# Example

```js id="g27"
console.log(1);

setTimeout(() => console.log(2));

Promise.resolve().then(() => console.log(3));

console.log(4);
```

Output:

```text id="g28"
1
4
3
2
```

Because:

* microtasks run before macrotasks

---

# 9️⃣ Promises

Represents future async value.

States:

* pending
* fulfilled
* rejected

---

# Example

```js id="g29"
fetch("/api")
  .then(res => res.json())
  .catch(console.error);
```

---

# 🔟 async/await

Syntactic sugar over promises.

---

# Example

```js id="g30"
async function getData() {
  const res = await fetch("/api");
}
```

---

# 🚨 Important

`await` pauses ONLY inside async function.

Does NOT block JS thread.

# 1️⃣5️⃣ `this` Keyword

Depends on HOW function is called.

---

# Regular Function

```js id="g37"

```

`this = obj`

---

# Arrow Function

Does NOT have own `this`.

Captures lexical `this`.

---

# 🚨 React Relevance

Arrow functions avoid manual binding.
